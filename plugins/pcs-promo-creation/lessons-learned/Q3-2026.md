## 1. Cross-vendor pipeline lessons (apply to every vendor)

### Kit build QA — run all 6 on every NS Create sheet before import
1. **Vertical P21 identical on every member row** of a kit (lead + details share the exact
   CA-Link/P21). A qty fix that updates only the lead's `[n]` → `[6]/[1]/[1]` mismatch
   (Milwaukee 7/16).
2. **Existing-kit match is order-independent** — compare by member **multiset** (SKU+qty),
   never the exact P21 string; the master may store members in a different order
   (Makita Q3: 34 already existed, 4 caught only by multiset).
3. **No split members** — a SKU appearing 2+ times → one line, qty N (`2630-20[2]`, not two
   `[1]`s). A split usually also signals a wrong free total (Milwaukee built qty-3 free when
   the deal was 2). "Buy 2 of the same" = one `[2]` line.
4. **Full price check** — kit `Sales` = Σ(paid-member **deck MAP** × qty); free goods = 0.
   The Kit Support export's `Reg` is list price, **not** the promo MAP — `Reg ≠ MAP` is
   expected, not an error.
5. **eBay-record requirement** — every member of a non-credit/non-RSA kit must resolve to an
   **EBAY** record; RSA/POS-redemption/credit kits use **regular** SKUs.
6. **Encoding** — UTF-8 **with BOM**; scan for CP1252 (`0x99` ™, `0x92` ') and `â€/Â`
   mojibake (DeWalt Q3 shipped 4 raw `0x99`).

### Shopify
- **Variant SKU = NetSuite internal ID**; search `sku:<id>` exact (plain text = noise).
- **Bulk tagging = one call.** Use `shopify_bulk_apply_tags` with an **inline `assignments`**
  array (`[{product_id, tag}, …]`, mixed tags OK) + `operation` + `store` → one API write +
  one rollback. **Don't** loop `shopify_add_product_tag` per product (token burn) and
  **don't** use `assignments_file` (server-side path the hosted sandbox can't read — the old
  "bulk fails" symptom). Build the ID list from `shopify_fetch_products` (writes a file +
  preview), not by paging `shopify_search_products` into context.
- **Duplicate-SKU-per-product (RTS/MTS):** reused kits often have **two** Shopify products
  sharing one SKU (older + newer). Tag only the **newest-created**; strip the tag from the
  older or it double-lists in the smart collection. TUP had none. Re-search late-built SKUs
  before declaring "not found" (Milwaukee 807784 appeared next day).
- **Smart collections:** rule enums UPPERCASE `{"column":"TAG","relation":"EQUALS",
  "condition":<tag>}`. Create with **no description** + **published:true**;
  `update_collection` can't edit the body (delete+recreate to change it).
- **Store set is vendor-dependent** — resolve via `list_stores`, don't hardcode. TUP =
  `toolupstore`, MTS = `toolup-my-tool-store`; ATO/DeWalt = `the-dewalt-store`; **Milwaukee
  uses `the-milwaukee-store` (RTS)** in place of ATO. Expect per-store catalog misses; report
  unmatched, don't force. **Never change product status** (kits stay DRAFT) — tag-only writes.

### Jira (PROM = "TU - Promotions", cloud `e45ae1ec-…`)
- Per-vendor **status column**: DeWalt work = status "DeWalt"; Makita = status "Makita"
  (id 10222). Epics: DeWalt **PROM-357**, Makita **PROM-356**.
- Custom fields: `11070` NetSuite Search (URL) · `11071` Product Tag (plain text) ·
  `11072` Collection Links (ADF). Put promo info in **these fields, not the description**;
  clear a rich-text field with empty ADF `{"type":"doc","version":1,"content":[]}` (null
  rejected).
- **Collection Links = front-end storefront URLs, not admin.** One store per line
  (hardBreak), `TUP: <url>` / `<brand-store>: <url>` / `MTS: <url>`, each URL as visible text
  **and** a link mark. Domains: `https://toolup.com`, `https://www.mytoolstore.com` (note
  `www.`), Milwaukee RTS `https://redtoolstore.com`; URL = `<domain>/collections/<handle>`.
- **Attachments can't be uploaded from the hosted sandbox** (proxy 403 to Atlassian; scoped
  token has no email/site) and the connector can't attach files — upload in Jira manually
  (or browser w/ explicit consent); the connector can only add a comment cross-reference.
- **Promo windows: trust the PROM `duedate`, not the deck art or a reused kit's Display
  Name.** (See pricing.)

### NLP / pricing (NetSuite is source of truth, not Shopify)
- Decode → paste into NLP Pricing Update revert search → export = the revert file →
  `build_workbook.py`. START tab = promo price into **both** `Online Sales Price` and
  `Promo MAP`; REVERT tab = restore the **true prior** OSP + Promo MAP (which **may differ**,
  and can be **$0** — flag $0 for human check, don't import blindly). Headers exactly
  `Internal ID, Online Sales Price, Promo MAP`; **add the ZZ `Name` column** as a standing
  default (`Internal ID, Name, Online Sales Price, Promo MAP`).
- **MAP compliance:** promo price = the floor (PMAPP). Items advertised **below** the floor
  are a violation risk; above = safe (start import steps them down). Carry-over already at/
  below the new promo = fine.
- **In-store-only PMAPP** rows (deck "ADVERTISING WINDOW: IN-STORE ONLY") are excluded from
  online NLP even though on master price lists (DeWalt Q3 pp.33–34, 60V Tool Connect+).
- **Existing/reused-kit repricing** is its own path: decode the changing internal IDs → revert
  search → NLP-format START/REVERT workbook keyed by **Internal ID** (kit revert exports have
  blank `Vendor Name`, so SKU matching fails — join on Internal ID).

### Calendar
- All-day; `colorId` 10 green = go-live/START, 11 red = takedown/REVERT; reminder popup at 0;
  TZ America/Los_Angeles. **Weekend/holiday date → schedule the Friday before** (Nathan's
  standing rule; mind Labor Day Mon 9/7/2026).

## 2. Promo-style additions seen this cycle

- **Milwaukee "free good with purchase" family** — anchor sells at its deck **Promo IMAP**;
  free good shows "**FREE**" (not a price). Variants → 10 P3 tags (M18 tool+5.0 batt, M18 bare
  tool+good, M18 combo+tools, M12 tool+batt, M18 tool+batt, M18 tool+good, Force Logic+good,
  OPE+good, accessory+good).
- **Buy Battery 2-Pack Get Free Good (HERO)** — anchor is a battery 2-pack at a **flat hero
  price** (M12 XC5.0 = $199, XC4.0 = $149), price in the page hero, not a line cell.
- **Buy Combo Kit Get 2 Select Free Tools (HERO)** — combo anchor at its IMAP + 2 free tools
  (Milwaukee 2695-24 $549 / 2696-24 $599 / 2696-26 $799).
- **Buy 2 Bare Tools (any mix) Get Free Good** — parser HOLDS as `buy2anymix`; build
  combinatorially = C(n,2)+n (all pairs + same-tool doubles), free good = 0.00 member, anchor
  = highest-MAP paid tool, Sales = sum of the 2 paid MAPs (Makita Q3 p34 LXT 30→465, p11 XGT
  10→55, p39 nailers 5→15). Confirm sibling free goods are built in NetSuite first.
- **e-Rebate "Kit" pages (NOT a kit build)** — "buy eligible kit, free good via eRebate": buy
  items are existing products, free good is mail-in → Other-Promotions/Jira, **no in-box kit**
  (Makita Q3 p23 Vacuum, p24 XGT OPE "M").

## 4. Gotchas (symptom → cause → fix)

- Transaction search 0 rows w/ item DECODE → used `{internalid}` (txn id) → use
  `{item.internalid}` + `Main Line = No`.
- `Read`/`pdftoppm` fails on deck → render with PyMuPDF (fitz) ~140 dpi to PNG, then read PNG.
- Large tool output blows context → save JSON, parse with jq/python, minimal fields, delegate
  big Shopify loops to a subagent.
- Tagging token burn / "bulk_apply_tags fails" → don't loop single-tag; don't use
  `assignments_file`; use inline `assignments` (§1 Shopify).
- Excel/OneDrive file lock on copy → write a new versioned filename (v2/v3).
- Duplicate kit member import error → one line, qty N (incl. a reg + TUP DCB205 dedup).
- Milwaukee deck price = **last of the 5-col table** (`50% OFF · HEAVY DUTY · PROMO PRICE ·
  IMAP · PROMO IMAP`) = Promo IMAP; free goods show "FREE".
- Deck-text OCR **row offset** → a single grep line can misattribute a price (2855P-20 looked
  $299, is $249); verify against the aligned block. Letter-suffix SKUs (`2855P-20`, `2565P-20`)
  break `\d{3,4}-\d{2}` regexes.
- Two windows on one deck → group by **per-promo** dates (Jira duedate authoritative), not one
  deck-level window.
- NetSuite **Display Name stale on reused kits** (shows old window/"RELIST"/"Flash Sale") →
  never derive the current window from it.
- Revert search returns **Promo MAP $0** for some kits → real value; flag before takedown
  import.
- **buy-2-any-mix** built only partially → after a parse, scan For-Review for **every**
  `buy2anymix` page; build each combinatorially; confirm sibling free goods are built first.
- Combinatorial member with **no MAP** → blank Sales on every pair using it → get MAP manually
  (Makita XGC01ZB = $309) and recompute.
- Coworker export may omit `Other-Promotions.csv` though the audit counts rows → reconcile via
  Parser-Audit and re-export if the Jira step needs it.
- Redaction: never print token files; check key presence only, never echo values.
